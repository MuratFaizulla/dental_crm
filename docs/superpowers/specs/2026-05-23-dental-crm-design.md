# Dental CRM — Техническое задание

**Дата:** 2026-05-23  
**Статус:** Согласовано  
**Язык системы:** Русский  
**Часовой пояс:** Asia/Almaty (Казахстан)

---

## 1. Общее описание

CRM-система для стоматологической клиники. Управляет пациентами, врачами, расписанием приёмов, медицинскими картами и финансами. Три типа пользователей с разными уровнями доступа: администратор, врач, пациент.

---

## 2. Архитектура

### Стек
- **Бэкенд:** Python 3.11 / Django 5.0.6 / Django REST Framework
- **Фронтенд:** React + TypeScript, React Router, Zustand (стейт), TanStack Query (запросы), Tailwind CSS
- **База данных:** PostgreSQL
- **Аутентификация:** JWT (SimpleJWT)
- **Очереди задач:** Celery + Redis (для уведомлений и автосообщений)
- **Файловое хранилище:** локальное или S3-совместимое (снимки, рентген, фото)

### Структура бэкенда (Django-приложения)
| Приложение | Назначение |
|---|---|
| `users` | Кастомная модель пользователя, JWT-аутентификация, роли |
| `doctors` | Врачи, ассистенты, специализации, услуги |
| `client` | Пациенты, ИИН, LTV, источник обращения |
| `records` | Записи на приём, расписание, типы оплаты |
| `medical` | Медкарты, зубная формула, план лечения, файлы |
| `finance` | Платежи, аналитика, отчёты по выручке |
| `notifications` | SMS/WhatsApp-уведомления, шаблоны, очередь |
| `api` | DRF ViewSets, сериализаторы, permissions |

### Структура фронтенда (React SPA)
Единое приложение с маршрутизацией по роли. После входа пользователь попадает в свой кабинет.

## 3. Роли и права доступа

### 3.1 Администратор
Полный доступ ко всем разделам системы.

**Расписание и записи:**
- Просмотр расписания в двух режимах: по врачу (колонки) и по клинике (все кресла)
- Переключение между видами одной кнопкой
- Создание новой записи к любому врачу
- Просмотр и редактирование записей любого врача

**Управление персоналом:**
- Добавление/редактирование профиля врача (ФИО, специализация, услуги)
- Назначение отпуска или больничного: врач блокируется на период, система предупреждает если есть существующие записи и предлагает перенести их
- Настройка рабочего графика врача (рабочие дни, часы приёма)
- Просмотр нагрузки врача

**Пациенты:**
- Список пациентов с поиском (по ФИО, ИИН, телефону)
- Карточка пациента: история визитов, LTV, источник обращения
- Добавление/редактирование пациента

**Финансы:**
- Список всех оплат с фильтрами (по дате, врачу, типу оплаты, статусу)
- Долги пациентов
- Рассрочка: фиксация долга и плана выплат прямо в системе

**Аналитика:**
- Выручка по периодам (день / неделя / месяц)
- KPI врачей: количество приёмов, выручка, средний чек
- LTV пациентов, конверсия, источники обращений

**Справочники:**
- Управление услугами, специализациями, статусами записей, типами оплаты, кресла/кабинеты

**Уведомления:**
- Ручная отправка SMS/WhatsApp любому пациенту
- Настройка шаблонов автосообщений

### 3.2 Врач
Доступ только к своим данным.

- **Мой день:** список своих приёмов на сегодня и на неделю
- **Мои пациенты:** полный список пациентов, которые были у этого врача
- **Медкарта пациента:** зубная формула, план лечения, история визитов, комментарии
- **Файлы:** просмотр и загрузка рентген-снимков, фото, документов для своих пациентов

### 3.3 Пациент (личный кабинет)
Доступ только к своей информации.

- **Мои записи:** предстоящие визиты (дата, врач, услуга)
- **История визитов:** прошлые приёмы с описанием услуг
- **Мои оплаты:** оплаченные счета, задолженности
- **Онлайн-запись:** самозапись к врачу через выбор специализации → врача → свободного времени
- **Уведомления:** просмотр полученных сообщений от клиники

## 4. Медицинская карта и план лечения

### 4.1 Зубная формула
- Визуальная схема зубов: верхний и нижний ряд по 16 зубов, расположенных по дуге
- Форма каждого зуба соответствует типу: моляр, премоляр, клык, резец
- Нумерация по системе FDI (11-18, 21-28, 31-38, 41-48)
- Цветовой статус каждого зуба:
  - Зелёный — здоров
  - Синий — пролечен
  - Красный — срочно лечить
  - Жёлтый — наблюдение
  - Серый — зуб отсутствует
- Клик по зубу открывает панель с деталями и кнопкой "Добавить в план лечения"

### 4.2 План лечения
- Список зубов с описанием: диагноз, рекомендованное лечение, срок
- Статус каждого пункта плана: не начато / в процессе / завершено
- После завершения лечения врач ставит галочку → зуб меняет цвет на синий (пролечен)
- Возможность указать причину откладывания (нет средств, пациент не готов) — для истории

### 4.3 Файлы пациента
- Рентген-снимки, фото, документы (согласия, договоры)
- Привязка файла к конкретному зубу или визиту
- Просмотр прямо в браузере (изображения) или скачивание

## 5. Расписание

### Два режима отображения
**По врачу:** временная сетка (строки = время, колонки = врачи), клик на ячейку = новая запись.

**По клинике:** временная сетка (строки = время, колонки = кресла/кабинеты), показывает загрузку по помещениям.

### Запись на приём
Форма создания записи содержит:
- Пациент (поиск по ФИО/телефону или создание нового)
- Врач + специализация
- Дата и время (из доступных слотов)
- Услуга, кресло
- Тип записи (первичный / повторный)
- Заметки

### Блокировка врача
Когда врач уходит в отпуск/на больничный:
1. Администратор указывает период недоступности
2. Слоты врача блокируются в расписании
3. Если есть существующие записи — система выводит список и предлагает перенести каждую

## 6. Уведомления и автосообщения

### Каналы
- SMS (через SMS-шлюз, например SMSC.ru или Kazatel)
- WhatsApp Business API (более популярен в Казахстане)

### Типы уведомлений
| Триггер | Сообщение |
|---|---|
| Запись создана | "Вы записаны к врачу [Имя] [дата] в [время]. Стоматология [Название]." |
| За 24 часа до приёма | "Напоминание: завтра в [время] визит к [врач]. Стоматология [Название]." |
| Врач поставил диагноз | "По результатам осмотра зуб №[X] требует лечения. Рекомендуем записаться до [дата]." |
| Зуб помечен как пролечен | "Зуб №[X] успешно пролечен! Следующий плановый осмотр через [срок]." |
| Через 3/6 месяцев (наблюдение) | "Прошло [срок]. Время проверить зуб №[X]. Запишитесь на осмотр." |
| Задолженность | "Напоминание: у вас задолженность [сумма]. Свяжитесь с нами." |

### Шаблоны
- Администратор редактирует шаблоны сообщений через справочники
- Переменные: `{patient_name}`, `{doctor_name}`, `{tooth_num}`, `{date}`, `{amount}` и т.д.
- Логи отправленных уведомлений хранятся в системе

## 7. Финансы

- Каждая запись на приём имеет: услуга, стоимость, скидка, итого, тип оплаты, статус оплаты
- Типы оплаты: наличные, карта, перевод, рассрочка
- Рассрочка: фиксируется общая сумма, уже оплаченная сумма, оставшийся долг, дата следующего платежа
- Финансовые отчёты: выручка по периодам, разбивка по врачам, разбивка по услугам
- Интеграция с кассовым ПО (1С, Kassanova и т.п.) — отдельный этап в будущем

## 8. API (Django REST Framework)

### Аутентификация
- JWT через `djangorestframework-simplejwt`
- Эндпоинт: `POST /api/token/` — получение пары access/refresh токенов
- Роль передаётся в JWT payload, фронтенд маршрутизирует по роли

### Права доступа (Permissions)
- `IsAdmin` — только администратор
- `IsDoctor` — врач (только свои данные)
- `IsPatient` — пациент (только своя информация)

### Основные эндпоинты (группы)
```
/api/v1/auth/          — вход, обновление токена
/api/v1/users/         — управление пользователями (admin only)
/api/v1/doctors/       — врачи, услуги, специализации
/api/v1/clients/       — пациенты
/api/v1/records/       — записи на приём, расписание
/api/v1/medical/       — медкарты, зубная формула, план лечения, файлы
/api/v1/finance/       — платежи, отчёты
/api/v1/notifications/ — уведомления, шаблоны
```

## 9. Этапы разработки

### Этап 1 — Ядро системы
- Настройка DRF, JWT, ролевые permissions
- API для врачей, пациентов, записей на приём
- Базовый React: вход, маршрутизация по роли, кабинет администратора (расписание + записи + пациенты)

### Этап 2 — Медкарта
- Модели и API для зубной формулы, плана лечения
- Интерактивная зубная формула на React (SVG)
- Загрузка и просмотр файлов (рентген, фото)

### Этап 3 — Кабинет врача
- Экран "Мой день"
- Работа с медкартами и планом лечения
- Загрузка файлов

### Этап 4 — Уведомления
- Настройка Celery + Redis
- Интеграция SMS/WhatsApp
- Шаблоны, триггеры, логи

### Этап 5 — Финансы и аналитика
- Отчёты по выручке и KPI врачей
- Рассрочка и долги
- Дашборд аналитики

### Этап 6 — Личный кабинет пациента
- Онлайн-запись
- История, оплаты, уведомления

### Этап 7 — Интеграции (будущее)
- Интеграция с кассовым ПО (1С, Kassanova)
- WhatsApp-бот для самозаписи
- Мобильная адаптация / PWA
