## Что есть в проекте

### Модули (Django-приложения)

| Приложение | Описание |
|-----------|----------|
| `users`   | Кастомная модель пользователя. Вход по email (не username). |
| `client`  | Пациенты — ФИО, ИИН, телефон, пол, дата рождения, источник обращения, LTV, врач. |
| `doctors` | Врачи, ассистенты, услуги, специализации. |
| `records` | Записи на приём — расписание, оплата, кабинет, статусы. |
| `api`     | Заготовка для REST API (пустой модуль). |

### Функциональность

- **Пациенты (Client)** — полная карточка: ФИО, ИИН (12 символов), телефон, пол, дата рождения, как узнал о клинике, привязка к врачу, LTV.
- **Врачи (Doctors)** — ФИО, специализация, услуги, описание.
- **Ассистенты (Assistant)** — ФИО, привязка к услуге.
- **Специализации (Specialization)** — название, стоимость.
- **Услуги (Service)** — название, привязка к специализации.
- **Записи (Record)** — полная запись приёма: пациент, врач, ассистент, услуга, зуб, стоимость, скидка, итог, дата и время, тип записи, кабинет, заметки, причина, тип оплаты, статус оплаты.
- **LTV (Lifetime Value)** — ведение истории трат пациента.
- **Классификаторы** — пол, источник обращения, статус LTV, статус записи, тип записи, кабинеты, типы и статусы оплаты.
- **Пользователи** — вход по email, кастомный менеджер, управление через Django Admin.

### Интерфейс

Вся работа через **Django Admin** (`/admin/`):
- Фильтрация, поиск, сортировка по всем ключевым полям.
- Иерархия по датам.
- Автодополнение для связанных полей.

## Как запустить проект

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd dental_crm
```

### 2. Создать и активировать виртуальное окружение

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux / macOS
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

> Также нужно установить psycopg2 для PostgreSQL:
> ```bash
> pip install psycopg2-binary
> ```

### 4. Создать базу данных PostgreSQL

```sql
CREATE DATABASE dental_base;
CREATE USER dental WITH PASSWORD 'dental123';
GRANT ALL PRIVILEGES ON DATABASE dental_base TO dental;
```

### 5. Применить миграции

```bash
cd src
python manage.py migrate
```

### 6. Создать суперпользователя

```bash
python manage.py createsuperuser
```

> Вход по **email** (не по username).

### 7. Запустить сервер

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Запуск через Docker

```bash
# Собрать образ
docker build -t dental-crm .

# Запустить контейнер
docker run -p 8000:8000 dental-crm
```

> Убедитесь, что PostgreSQL доступен из контейнера (настройте DATABASE_URL или docker-compose).

## Доступ к админ-панели

После запуска перейдите по адресу:

```
http://127.0.0.1:8000/admin/
```

Войдите с данными суперпользователя, созданного командой `createsuperuser`.

## Известные проблемы

| Проблема | Описание |
|---------|----------|
| `UserLoginForm` | Ссылается на поле `username`, которого нет — нужно переписать под `email` |
| `ProfileForm` | Ссылается на поле `image`, которого нет в модели User |
| `ALLOWED_HOSTS` | Пустой список — нужно добавить хосты для деплоя |
| Секреты в настройках | `SECRET_KEY` и данные БД захардкожены — нужен `.env` файл |
